## 摘要
本文提出了FusionCIM，一种算子融合驱动的存内计算加速器架构，用于高效且可扩展的大语言模型推理，具有三项关键创新：（1）混合存内计算流水线架构，将QK^T计算映射到基于内积的存内计算（IP-CIM）上，PV聚合映射到基于外积的存内计算（OP-CIM）上，实现高效矩阵乘法融合；（2）QO驻留数据流，消除了CIM中重复的KV加载以及转置融合下缓冲区中的K矩阵访问，显著提高了芯片上的数据复用；（3）模式感知的在线softmax机制，利用注意力分数的分布规律减少非线性融合中的指数缩放开销。在LLaMA-3模型上的实验结果表明，与先前最先进的基于CIM的设计相比，FusionCIM在系统级实现了高达3.86倍的能耗节省和1.98倍的加速，能效达29.4 TOPS/W。

## Abstract
In this paper, we propose FusionCIM, an operator-fusion-driven compute-in-memory (CIM) accelerator architecture for efficient and scalable LLM inference, with three key innovations: (1) a hybrid CIM pipeline architecture that maps QKT computation on inner-product-based CIM (IP-CIM) and PV aggregation on outer-product-based CIM (OP-CIM) for efficient matrix multiplications fusion; (2) a QO-stationary dataflow that eliminates repeated KV loading in CIM and K-matrix access in buffer under transpose fusion, significantly improving data reuse on chip; and (3) a pattern-aware online-softmax mechanism that exploits distribution regularities of attention scores to reduce exponential rescaling overhead for non-linear fusion. Experimental results on LLaMA-3 model show that FusionCIM achieves up to 3.86x energy saving, and 1.98x speedup compared with prior SOTA CIM-based designs with 29.4 TOPS/W energy efficiency at the system level.

# FusionCIM 论文详细总结

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：大型语言模型（LLM）推理面临巨大的计算和内存访问开销，尤其在长上下文场景下，注意力机制中的激活矩阵（如分数矩阵）访问成为主要瓶颈。现有计算‑存内（CIM）加速器虽能减少权值搬运，但存在三大挑战：
  1. 频繁的片外激活访问（分数矩阵大小为 O(N²)）；
  2. 高片内 KV 访问：分块算法导致 KV 矩阵被重复加载到 CIM 阵列，且 K 矩阵转置需要额外缓冲及访问开销；
  3. 在线 softmax 算法中的频繁指数缩放操作增加了非线性计算开销。
- **整体含义**：提出 FusionCIM，通过算子融合驱动的架构、优化的数据流和模式感知调度，同时减少片外、片内存储访问和非线性计算开销，实现高效可扩展的 LLM 推理。该工作证明了 CIM 架构在 LLM 推理中通过融合设计可大幅超越传统方案。

## 2. 论文提出的方法论：核心思想、关键技术细节

**核心思想**：基于内积 CIM（IP‑CIM）和外积 CIM（OP‑CIM）的混合架构，配合 QO‑驻留数据流和模式感知在线 softmax，实现注意力计算的全流水线融合。

**关键技术细节**：

- **混合 CIM 流水线架构**：将 QKᵀ 矩阵乘法映射到 IP‑CIM，PV 聚合映射到 OP‑CIM，中间软连接 SoftMax 模块。IP‑CIM 以 Q 为驻留权值，K 为输入流；OP‑CIM 以 O（输出部分和）为驻留值，经 softmax 后的 P 和 V 为输入流。此设计避免了传统方案中 KV 作为静态权值反复加载的问题。
- **QO‑驻留数据流**：Q 仅从片外加载一次，在 IP‑CIM 中保持不动；输出 O 的部分和直接在 OP‑CIM 中累积，无需额外搬运。K 和 V 以向量流形式逐列输入，同时实现**原位 K 转置**（无需额外转置缓冲区），极大减少片内读写。
- **模式感知在线 softmax**：利用注意力分数分布中行最大值常出现在对角附近的特点，调度内循环从对角位置开始。通过**交错分片**（inter‑tile）和**反向索引**（intra‑tile），提早确定行最大值，从而减少在线 softmax 中对输出部分和的指数缩放次数。在 LLaMA‑3 上可消除 61.4% 的缩放操作。
- **两级流水线**：向量级流水（QKᵀ → SoftMax → PV）和位级流水（8‑bit K/V 按位串行，与 8‑step Taylor 指数计算对齐），保证全流水无气泡。
