## 摘要
大型语言模型越来越多地部署在功耗和面积预算紧张的边缘设备上。虽然混合精度通用矩阵乘法(GEMM)降低了算术复杂度，但量化推理的主要开销往往来自于反量化操作和非线性算子。基于查找表(LUT)的方法通过预计算输出并用查表替代重复算术来缓解这些开销，但现有设计会带来显著容量和查表延迟开销。本文提出PALUTE，一种基于查找表的存内处理加速器，它构建在单片三维DRAM之上，用于高效的边缘大语言模型推理。PALUTE支持DRAM内查表操作，利用M3D DRAM存储阵列瓦片(tile)的垂直组织方式，以低面积开销实现高并行性。近存储LUT生成器支持对GEMM和逐元素一元非线性算子进行低延迟LUT生成，同时系统级的分层与调度策略最大限度地减少跨内存层级的数据移动。使用周期精确仿真和RTL综合的评估表明，PALUTE在0.16W功耗下实现了1264 TPS的端到端吞吐量，与CHIME相比能效提高了12.8倍，与FIGLUT相比提高了1.6倍，在Qwen3-4B模型的W4A4配置下，面积效率比PIMPAL提高了2.0倍。

## Abstract
Large language models are increasingly deployed on edge devices with tight power and area budgets. While mixed-precision GEMM reduces arithmetic complexity, quantized inference is often dominated by dequantization and nonlinear operators. Lookup Table (LUT)-based method mitigates these costs by precomputing outputs and replacing repeated arithmetic with table lookups, but existing designs incur significant capacity and lookup-latency overheads. This paper presents PALUTE, a LUT-based Processing-In-Memory accelerator built on Monolithic 3D DRAM for efficient edge LLM inference. PALUTE enables in-DRAM LUT queries that exploit the vertical organization of M3D DRAM memory array tiles to achieve high parallelism with low area overhead. A near-memory LUT generator supports low-latency LUT generation for both GEMM and element-wise unary nonlinear operators, while a system-level tiering and scheduling strategy minimizes data movement across memory tiers. Evaluation using cycle-accurate simulation and RTL synthesis shows that PALUTE achieves 1,264 TPS end-to-end throughput at 0.16 W, improving energy efficiency by 12.8$\times$ over CHIME and 1.6$\times$ over FIGLUT, improving area efficiency by 2.0$\times$ over PIMPAL under W4A4 across Qwen3-4B models.

- **核心问题**：大型语言模型（LLM）在边缘设备部署时面临严格的功耗、面积和内存带宽约束。虽然混合精度量化（如W4A4）降低了算术复杂度，但实际推理中反量化操作和非线性算子（如Softmax、GELU）成为主要性能瓶颈，占运行时比例高达20%-90%。传统纯算术优化无法解决数据移动导致的“内存墙”问题。
- **研究动机**：查找表（LUT）方法通过预计算输出值，用查表替代重复算术运算，可完全消除反量化开销。然而，现有LUT加速器（如pLUTo、PIMPAL、FIGLUT、CHIME）受限于DRAM行缓冲/突发接口的并行度限制、LUT容量开销大、单次查询延迟高、或未实现端到端流水线等问题。
- **整体含义**：本文提出PALUTE，一种基于单片3D DRAM（M3D DRAM）的存内处理加速器，利用其垂直存储阵列瓦片（MAT）实现高并行LUT查询，同时集成近内存LUT生成器和系统级数据分层调度，以极低功耗（0.16W）和较小面积开销实现高效的边缘LLM推理，展示了LUT范式在存内计算中的实用路径。

## 2. 论文提出的方法论

- **核心思想**：在M3D DRAM内直接执行LUT查询，利用垂直比特线（VBL）组织方式实现大规模并行；将GEMM和逐元素一元非线性算子（如GELU）映射为LUT操作，避免反量化和传统乘法累加运算。
- **关键技术细节**：
  - **LUT查询机制**：
    - GEMM分解：将长点积拆分为小段（segment），例如每个段长度为2，权重为4位量化（INT4），则每个激活段对应有限个权重组合（256种可能），预计算所有可能的部分和并存入LUT。运行时以权重为索引直接查表。
    - 非线性算子：对GELU等函数，将输入量化值映射为LUT索引，直接读取预计算输出。
    - **半表技术（Half-Table）**：利用符号对称性，只存储非负索引对应的LUT值，负值通过翻转符号位映射到对称索引，读取后恢复符号，将LUT容量减半。
  - **M3D DRAM内的垂直MAT LUT查询**：每个MAT内，水平字线（WL）存储LUT索引，垂直位线（BL）存储LUT值；多层MAT堆叠，每个MAT底部的SA（感测放大器）下方集成匹配逻辑（行解码器）和开关，实现一次扫查同时读取多个LUT（并行度高达8,388,608个并发查询）。每个MAT可存储多个LUT副本，进一步提高吞吐。
  - **近内存LUT生成器**：位于逻辑die，通过混合键合（hybrid bonding）与DRAM tier连接，提供约1.38 TB/s带宽。每个生成器支持GEMM核和逐元素一元核，能在3个时钟周期内为给定激活段生成全部LUT条目（利用MEGA乘法器和MEGA加法器阵列并行计算）。
  - **系统级数据分层与调度**：根据M3D DRAM中各层字线延迟非均匀（下层字线寄生更大），将访问频繁的权重和初始token放置在靠近逻辑die的低层（延迟低），LUT放在中间层，KV缓存放在高层（靠近累加器）。调度流程：权重和初始token通过混合键合载入→逻辑die的LUT生成器构建LUT→写入DRAM→以权重为索引进行LUT查询→结果送至bank级累加器→KV缓存读写。
