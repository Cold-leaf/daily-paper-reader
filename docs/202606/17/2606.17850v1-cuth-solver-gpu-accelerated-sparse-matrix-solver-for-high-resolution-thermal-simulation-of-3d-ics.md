## 摘要
粗粒度热仿真往往会低估局部热问题，可能遗漏关键热点。因此，精确分析需要细粒度信息，这会大幅增加网格分辨率，从而增大计算工作量。幸运的是，系数矩阵通常是稀疏的，且具有规则的稀疏模式，这提供了优化机会。然而，GPU上现有的通用矩阵求解器很少利用这些领域特定属性，从而在数据存储、内存访问、并行性、计算效率和硬件利用率方面遇到瓶颈。为此，我们提出了CUTh-Solver，一个协同设计的基于GPU加速的预处理共轭梯度法（PCG）稀疏求解器框架，用于求解高分辨率稳态和瞬态3D IC热仿真中出现的对称正定（SPD）系统。在数据存储方面，CUTh-Solver压缩对角（DIA）存储格式以去除冗余。为了优化内存访问，CUTh-Solver采用按对角线的SpMV实现合并内存访问。我们进一步发现并行性和预处理质量之间存在关键冲突，因此采用了一种高并行性的预处理策略。为了提高计算效率和硬件利用率，我们采用了一种自适应细粒度混合精度策略，利用多种浮点单元避免资源竞争，从而在保证数值稳定性的前提下提高吞吐量。实验结果表明，与GPU加速的COMSOL Multiphysics 6.4相比，CUTh-Solver实现了高达25.8倍的加速，与NVIDIA的原生通用库（AmgX、cuSPARSE、cuDSS）相比，加速超过3倍。消融实验验证了每个优化的单独贡献。代码可在https://github.com/Chenghan-Wang/CUTh-Solver获取。

## Abstract
Coarse-grained thermal simulation tends to underestimate localized thermal issues, potentially missing critical hotspots. Accurate analysis, therefore, demands fine-grained information, which dramatically increases grid resolution and thus computational workload. Fortunately, the coefficient matrices are often sparse with regular sparsity patterns, offering optimization opportunities. However, existing general-purpose matrix solvers on GPUs rarely exploit these domain-specific properties, thereby encountering bottlenecks in data storage, memory access, parallelism, computational efficiency, and hardware utilization. Therefore, we propose CUTh-Solver, a co-designed GPU-accelerated Preconditioned Conjugate Gradient (PCG)-based sparse solver framework for Symmetric Positive Definite (SPD) systems arising from high-resolution steady-state and transient 3D IC thermal simulation. For data storage, CUTh-Solver condenses the Diagonal (DIA) storage format to remove redundancy. To optimize the memory access, CUTh-Solver employs diagonal-wise SpMV to achieve coalesced memory access. We further observe a critical conflict between parallelism and preconditioning quality and thus adopt a high-parallelism preconditioning strategy. To improve computational efficiency and hardware utilization, we employ an adaptive fine-grained mixed-precision strategy that leverages diverse floating-point units to avoid resource contention, enhancing throughput without compromising numerical stability. Experimental results show that CUTh-Solver achieves up to 25.8x speedup over GPU-accelerated COMSOL Multiphysics 6.4 and over 3x speedup over NVIDIA's native general-purpose libraries (AmgX, cuSPARSE, cuDSS). Ablation studies validate the individual contribution of each optimization. The code is available at: https://github.com/Chenghan-Wang/CUTh-Solver

#### 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究背景**：随着摩尔定律的极限，三维集成电路（3D IC）成为重要解决方案，但其高集成密度带来了严峻的散热问题。为了准确捕捉芯片上的局部热点，进行高分辨率的细粒度热仿真是必要的。然而，细粒度仿真会导致计算网格数量巨大，从而使求解线性系统 `Ax = b` 成为整个仿真流程的计算瓶颈，通常耗时超过60%。
- **核心问题**：虽然3D IC热仿真生成的矩阵是稀疏且具有规则模式（如7对角、对称正定SPD）的，但市面上的通用GPU稀疏求解器（如NVIDIA cuSPARSE, AmgX, cuDSS）未能充分利用这些领域特定属性，因此在以下几个方面存在效率瓶颈：
    1.  **数据存储**：通用格式存储冗余，未利用矩阵的对称性和带状特性。
    2.  **内存访问**：稀疏矩阵向量乘（SpMV）通常导致不规则的内存访问，延迟高。
    3.  **并行性**：CPU上高效的预条件子（如不完全Cholesky）在GPU上因复杂的依赖关系而并行度受限。
    4.  **硬件利用率**：现代GPU的FP64单元稀缺，而通用求解器默认使用FP64，导致计算单元闲置，利用率低。
- **研究意义**：针对上述问题，本文提出CUTh-Solver，一个为3D IC热仿真量身定制的GPU加速稀疏求解器，旨在通过领域知识驱动的方法，显著提升求解效率，从而加速整个热仿真过程。

#### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：利用3D IC热仿真中产生的**7对角、对称正定（SPD）** 矩阵的领域先验知识，对数据存储、内存访问、并行策略和计算精度进行协同优化，实现求解器性能的全面提升。
- **关键技术细节**：
    - **技术1：Quad-Diag 存储格式**
        - 核心：压缩传统DIA格式，去除冗余。利用矩阵的对称性，仅存储上三角部分的4条对角线（D4, D5, D6, D7）。同时，解耦了对角线间的“填充”对齐问题，节省存储空间。每个对角线存储为FP64、FP32和FP16三个副本（A1, A2, A3），为混合精度策略做准备。
    - **技术2：基于对角线的SpMV（合并内存访问）**
        - 核心：将传统的行方向SpMV计算转换为列方向（对角线方向）的多段连续元素乘法和向量加法。由于访问操作数向量x时，每个对角线访问的是一个连续的子区间，因此大量DRAM访问是合并的，大幅降低了访问延迟。
    - **技术3：高并行性预条件（Chebyshev多项式预条件）**
        - 核心：GPU上，CPU友好的预条件子（如IC, AMG）因复杂的依赖和递归计算而并行度不足，甚至不如不用预条件。因此，采用Chebyshev多项式预条件，其核心计算仅分解为一系列高并行度的SpMV和AXPY操作。
        - **协同效应**：这种方法使得SpMV成为PCG中的绝对主导内核，最大化利用Quad-Diag格式和SpMV优化；同时，预条件阶段对数值精度不敏感，为混合精度策略提供了空间。
    - **技术4：自适应细粒度混合精度策略**
        - 动机：FP64算力稀缺，而低精度（FP16/32）算力充足。但直接使用低精度会导致数值漂移甚至发散。
        - 实现：该策略在**两个维度**上动态调整精度，并由**安全机制**控制切换。
            1.  **阶段（时间）维度**：早期阶段采用低精度加速，晚期切换到FP64保证收敛。
            2.  **组件（空间）维度**：对不同计算组件分配不同精度。例如，预条件阶段使用FP32；SpMV内核使用物理感知的FP16/32混合；内积和向量更新使用FP64。
            3.  **安全机制**：通过检测残差变化等标准判断切换时机，并在切换后进行“热重启”以清除累积的低精度误差。
