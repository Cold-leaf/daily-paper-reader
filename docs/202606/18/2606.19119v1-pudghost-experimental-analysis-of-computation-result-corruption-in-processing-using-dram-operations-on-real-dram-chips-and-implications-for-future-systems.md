## 摘要
处理中内存（PuD）是一种有前景的计算范式，通过同时多行激活（SiMRA）利用每个DRAM列作为计算引擎，减轻主存与处理单元之间的频繁数据移动。不幸的是，DRAM密度缩放可能阻碍PuD的优势：更密集的单元阵列使行和列更接近，使得常规DRAM操作容易受到邻近单元噪声和干扰的影响。然而，先前没有工作研究非参与计算的行或列的干扰是否会损害PuD的稳健性。在这项工作中，我们揭示了PuDGhost，一种干扰现象，其中给定列中的PuD操作由于来自1）未激活DRAM行中的数据和2）在同一SiMRA操作下同时计算的其他列中的数据的干扰而产生错误结果。PuDGhost违反了每列计算仅依赖于其自身操作数据的理想图景，威胁到未来的PuD系统。我们首次使用来自12个模组的96个真实DDR4 DRAM芯片对PuDGhost进行了广泛表征，量化了各种条件下的这两种干扰源。在我们的15个新经验观察中，我们强调两个主要结果：1）相邻未激活行中的数据对随机输入的SiMRA输出影响高达10%，2）同时计算的列中的数据对随机输入的SiMRA输出影响高达48%。基于这些发现，我们提出了跨PuD计算堆栈多层的对策。具体来说，我们在真实DDR4 DRAM芯片上评估了：1）稳健的列筛选，降低在PuDGhost存在下使用不可靠列的风险，以及2）计算行布局，通过在计算行之间设置专用行来缓解PuDGhost。我们的解决方案大大提高了PuD计算精度，并为未来稳健的PuD系统奠定了基础。

## Abstract
Processing-using-DRAM (PuD) is a promising computation paradigm that alleviates frequent data movement between main memory and processing units by using each DRAM column as a computation engine via simultaneous multiple-row activation (SiMRA). Unfortunately, DRAM density scaling may hinder PuD's benefits: denser cell arrays bring rows and columns closer, making regular DRAM operations susceptible to noise and interference from neighboring cells. Yet no prior work investigates whether interference from rows or columns not intended to participate in computation can compromise PuD robustness.   In this work, we reveal PuDGhost, an interference phenomenon where a PuD operation in a given column produces erroneous results due to interference from 1) data in non-activated DRAM rows and 2) data in other columns that compute concurrently under the same SiMRA operation. PuDGhost violates the ideal picture that each column's computation depends solely on its own operand data, threatening future PuD systems. We present the first extensive characterization of PuDGhost using 96 real DDR4 DRAM chips from 12 modules, quantifying these two interference sources under various conditions. Among our 15 new empirical observations, we highlight two major results: 1) data in adjacent non-activated rows affects SiMRA outputs by up to 10% for random inputs, and 2) data in concurrently computing columns affects SiMRA outputs by up to 48% for random inputs. Guided by these findings, we propose countermeasures across multiple layers of the PuD computing stack. Specifically, we evaluate on real DDR4 DRAM chips: 1) robust column screening that reduces the risk of using unreliable columns in the presence of PuDGhost, and 2) a compute row layout that mitigates PuDGhost via dedicated rows between compute rows. Our solutions greatly improve PuD computation accuracy and provide a foundation for robust future PuD systems.

# PuDGhost: 真实DRAM芯片上处理中内存操作计算结果损坏的实验分析

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **背景**：Processing-using-DRAM（PuD）是一种新兴计算范式，通过同时多行激活（SiMRA）利用DRAM阵列的模拟特性，使得每个列可作为独立计算单元，实现大规模并行且高能效的内存内计算。然而，随着DRAM密度持续提升，单元阵列更加密集，行与列之间物理间距缩小，导致常规操作已易受邻近单元噪声干扰。
- **核心问题**：第一，现有PuD研究假设每列SiMRA输出仅依赖于该列自身的操作数，但未考虑非参与计算的行或列的数据是否会影响结果。第二，DRAM中的隔离性本指一个内存访问不应影响其他存储单元，但PuD将DRAM从存储器转变为计算设备后，隔离性应扩展为：**一次计算的输出不应受非操作数数据影响**。第三，已知DRAM存在行锤（RowHammer）、列干扰（ColumnDisturb）等读干扰现象，但PuDGhost是**不同的机制**：它并非由重复激活引起，而是单次SiMRA操作中由邻近行/列电荷耦合导致的瞬时计算错误。
- **研究意义**：揭示PuDGhost现象对PuD可靠性的根本威胁，为设计未来鲁棒PuD系统提供实验基础和缓解方案。

## 2. 论文提出的方法论

### 核心思想
- 通过精细控制的FPGA测试平台，在真实商用DDR4 DRAM芯片上执行SiMRA操作，系统改变非操作数区域（非激活行、同时计算列）的数据模式，并量化其对SiMRA输出结果的影响。
- 定义**规范性输出1比例（Norm. po1）**：将目标列输出中逻辑1的比例与基线（所有非操作数数据为随机模式，pc=0.5）下的比例相除，以评估干扰方向和强度。

### 关键技术细节
1. **实验硬件**：使用DRAM Bender（基于Xilinx Alveo U200 FPGA）精确控制DRAM命令，配合热电偶和加热垫控制温度。
2. **实验对象**：96个SK Hynix DDR4芯片（12个模组，含不同密度4Gb/8Gb和die版本A/M），通过逆向工程确定物理行映射、子阵列边界、真/反单元类型、奇偶列。
3. **SiMRA验证**：通过APA（ACT-PRE-ACT）命令序列实现2/4/8/16/32行同时激活，并用后续WRITE验证哪些行被激活。
4. **两种干扰源实验**：
   - **非激活行干扰（RQ1）**：将邻近行、同一子阵列非邻近行、相邻子阵列等区域作为控制行，设置固定数据（全0/全1/随机/字节模式），测量po1变化。
   - **同时计算列干扰（RQ2）**：随机选取1/8列作为目标列，其余列（控制列）的SiMRA输入数据设为固定模式（全0/全1/随机），测量目标列输出偏差。
5. **缓解对策**：
   - **鲁棒列筛选**：CS-1在筛选阶段对邻近行采用多种数据模式（全0、全1或更精细扫描），只保留在所有模式下均可靠的列；CS-2在筛选和执行阶段均固定邻近行数据。
   - **隔离行布局**：在计算行组之间插入专用隔离行（存储固定数据），确保计算行邻近行数据在执行期间不变。

### 算法流程（文字说明）
1. 初始化：写入控制数据到所有非操作数区域（如邻近行、其他列）。
2. 写入随机操作数到SiMRA行。
3. 执行SiMRA（APA序列）。
4. 读取SiMRA输出，计算该列/所有列的po1。
5. 重复128次随机输入采样，取统计结果。

## 3. 实验设计

- **数据集/场景**：
  - 使用**96个真实DRAM芯片**（SK Hynix，来自12个模组，含3种产品类型：TimeTec 4Gb、TeamGroup 4Gb、SK Hynix 8Gb）。
  - 每个模组选取3个代表性子阵列（顶部/中部/底部），每个子阵列随机选择一组SiMRA行。
  - 激活行数：2、4、8、16、32。
  - 数据模式：全0、全1、随机（pc=0,0.25,0.75,1.0）、结构化字节模式（0x00,0xF0,0x55,0xAA,0xFF）。
  - 温度：50°C、60°C、70°C、80°C。
  - 空间属性：不同奇偶列组合（Opposite-Parity、Same-Parity）、列局部性分析。
- **基准（Baseline）**：所有非操作数数据为随机模式且pc=0.5。
- **对比方法**：
  - 无PuDGhost感知的筛选（Base）：筛选时邻近行固定一种随机模式，执行时动态变化。
  - CS-1-01：仅用全0和全1两种极端数据模式筛选。
  - CS-1-sweep：用5种pc值（0.0,0.25,0.5,0.75,1.0）筛选。
  - CS-2-0/CS-2-1/CS-2-check：筛选和执行时均固定邻近行为全0/全1/棋盘格。
  - 应用级：GEMV（矩阵-向量乘法）和TRNG（真随机数生成）对比Base-rand、Base-worst、CS-1、CS-2。

## 6. 论文的主要结论与发现

1. **非激活相邻行干扰**：邻近行存储全0（全1）会使SiMRA输出偏向逻辑0（逻辑1），偏差随邻近行中逻辑1比例单调增加。对于随机输入，影响高达10%。
2. **同时计算列干扰**：其干扰更强（最高48%），且与激活行数有关。对于2-4行激活，输出偏向单调；对于8行以上，在中间pc值出现峰值后反转（非单调）。
3. **奇偶列差异**：对位奇数列干扰单调，同位奇数列干扰非单调。
4. **温度影响较小**：非激活行干扰温度不敏感；同时计算列干扰随温度单调变化但幅度不大。
5. **输入依赖性**：只有储存逻辑1的输入位易受邻近行干扰，逻辑0位几乎不受影响。
6. **缓解效果**：
   - CS-2-1（固定邻行为全1）比无感知的Base降低125倍列失效率和91倍比特错误率，列通过率提升1.06倍。
   - 在GEMV应用中，NMSE降低达413倍（相较于Base-worst）。
   - 在TRNG中，固定数据可阻止93%的熵损失。

## 7. 优点

- **首次性**：首次在真实DRAM芯片上实验证明非操作数数据对PuD计算结果有干扰，填补了重要空白。
- **实验全面性**：覆盖96芯片、多种激活行数、多种数据模式、温度、空间属性，观察量丰富。
- **实用性**：提出了可直接实施的缓解方案（列筛选+隔离行布局），并在真实应用上验证了有效性。
- **结果真实可靠**：使用开放硬件测试平台（DRAM Bender），方法透明，可复现。
- **对领域的启发**：指出了现有PuD仿真模型（假设列间独立错误）的不足，并提出需考虑列间依赖性，推动未来解析模型发展。

## 8. 不足与局限

- **芯片制造商单一**：仅测试SK Hynix DDR4芯片，因其他品牌在COTS上不支持SiMRA。作者推测所有支持层次行解码器的芯片均可能存在PuDGhost，但未验证。
- **缺乏根本原因分析**：实验为现象学表征，未通过设备级（TCAD/SPICE）仿真验证电荷耦合的具体机制，仅提出假设。
- **缓解方案的权衡**：CS-1降低有效列数，CS-2引入隔离行面积开销（约0.68%），CS-2-1虽效果好但需固定邻行数据，可能不适用于所有计算模式。
- **未覆盖所有DRAM类型**：仅DDR4，未测试DDR5、LPDDR、HBM等，这些芯片可能具有不同布局和干扰特性。
- **应用评估有限**：GEMV和TRNG虽代表性，但未涉及更复杂PuD工作负载（如数据库、图计算）。误差累积模型未深入分析。
- **实验偏见风险**：虽控制变量严格，但随机输入模式可能未完全覆盖所有可能数据分布，尤其是极端稀疏模式。
