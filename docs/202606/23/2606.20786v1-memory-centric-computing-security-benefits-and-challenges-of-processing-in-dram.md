## 摘要
当今的计算系统以处理器为中心：它们需要处理元件（如CPU）与主内存（DRAM）之间频繁的数据移动，导致性能和能耗方面的显著低效。以内存为中心的计算则将计算移至数据处，在数据生成和存储的所有位置附近实现计算能力，从而大幅减少数据访问和移动的性能与能耗开销。从以处理器为中心到以内存为中心的范式转变，对系统安全产生了重要且未充分探索的影响。将内存从被动的、非活跃的存储转变为活跃的计算基底，为系统安全带来了优势与挑战：它可以提供新的内存内安全原语，并减少数据暴露，但也可能暴露新的攻击面。本文讨论了以内存为中心的计算的安全优势与挑战，特别是内存处理（Processing-in-DRAM，PiD）——一种利用并增强DRAM芯片操作特性以对存储在DRAM中的数据进行计算的范式。具体来说，我们描述了1）基于DRAM的最新真随机数生成器，其在真实DRAM芯片上提供高达16.05 Gb/s的吞吐量，以及物理不可克隆函数，其评估延迟比先前最先进方案降低5.75%；2）PiD的两个关键安全挑战：放大的DRAM读取干扰（例如，引发第一个比特翻转所需的最小DRAM访问次数减少158倍）和高吞吐量内存时序信道（例如，通信吞吐量为14.8 Mb/s）。我们认为，现在是时候将DRAM（以及一般内存）设计、使用和编程为一种结合计算、存储和安全的基底，其中计算能力、存储密度和安全性都是关键目标。

## Abstract
Today's computing systems are processor-centric: they require frequent data movement between processing elements (e.g., CPU) and main memory (DRAM), leading to significant inefficiencies in performance and energy consumption. Memory-centric computing instead moves computation to the data, enabling computation capability in and near all places where data is generated and stored, and greatly reducing the performance and energy overheads of data access and data movement. This shift from a processor-centric to a memory-centric paradigm has important and underexplored consequences for system security. Turning memory from a dumb, inactive store into an active computing substrate introduces benefits as well as challenges for system security: it can provide new in-memory security primitives and also reduce data exposure, but it can also expose new attack surfaces. This work discusses the security benefits and challenges of memory-centric computing, specifically Processing-in-DRAM (PiD), a paradigm where the operational characteristics of a DRAM chip are exploited and enhanced to perform computation on data stored in DRAM. Specifically, we describe 1) new state-of-the-art DRAM-based true random number generators that provide up to 16.05 Gb/s throughput and physical unclonable functions with 5.75% lower evaluation latency than the prior state-of-the-art, both on real DRAM chips and 2) two key security challenges of PiD: amplified DRAM read disturbance (e.g., 158x reduction in the minimum number of DRAM accesses required to induce the first bitflip) and high throughput memory timing channels (e.g., a communication throughput of 14.8Mb/s). We believe it is time to design, use, and program DRAM, and in general memory, not as an inactive storage substrate, but as a combined computation, storage, and security substrate, where computational capability, storage density, and security are all key goals.

### 论文核心问题与整体含义

- **研究背景**：当前计算系统以“处理器为中心”，数据需要在处理器和主存（DRAM）之间频繁移动，这导致了严重的性能瓶颈和巨大的能耗开销，尤其是在数据密集型应用（如AI、基因组分析）中。
- **核心问题**：如何从根本上解决数据移动带来的性能、能耗和安全性问题？以“内存为中心的计算”（MCC），特别是“处理进DRAM”（Processing-in-DRAM, PiD）范式被视为一种有前景的解决方案。然而，将原本被视为“惰性”存储单元的内存转变为主动的计算基底，会给系统安全带来一系列未被充分探索的**新机遇**（安全优势）和**新风险**（安全挑战）。
- **整体含义**：本文旨在系统性地探讨PiD范式的安全两面性。一方面，它揭示了如何利用PiD技术在商用DRAM芯片上高效、低成本地实现关键安全原语（如真随机数生成器和物理不可克隆函数）；另一方面，它警示了PiD操作（如多行激活）可能引发的读干扰放大问题和高吞吐量时序信道攻击，从而揭示出在享受计算便利性的同时，必须正视和解决的新安全威胁。

### 论文提出的方法论

该文并非提出一个单一的方法论，而是基于对真实DRAM芯片的实验，系统阐述了PiD在安全方面带来的**既有优势又有挑战的两面性**，分别对应了四个具体的工作。

- **安全优势：在DRAM内实现安全原语**
  1.  **核心思想**：利用违反DRAM标准时序参数（如`tRAS`）导致的非确定性行为来产生随机性或芯片特有的签名。
  2.  **SiMRA-TRNG（真随机数生成器）**：
      - **流程**：通过`ACT→PRE→ACT`指令序列，以缩短的时序**同时激活多行**（Simultaneous Multiple-Row Activation, SiMRA）。这些行中预存了数量相等的“0”和“1”，它们通过位线进行电荷共享，使位线电压被扰动到感测放大器的不稳定范围内，从而导致感测放大器随机地输出“0”或“1”。
      - **后处理**：提取的原始比特流经过SHA-256哈希函数以消除偏差和相关性，从而获得高质量的真随机数。
  3.  **SiMRA-PUF（物理不可克隆函数）**：
      - **流程**：同样利用SiMRA操作。在同时激活多行并发生电荷共享后，由于工艺制造差异，不同位线上的电荷共享结果会略有不同。对于那些稳定地偏向“0”或“1”的位线，其输出就可以作为设备独有的、可重复生成的响应签名。
      - **挑战-响应**：将`（bank, subarray）`对作为挑战，将SiMRA操作后的感测放大器输出作为响应。

- **安全挑战：新的攻击面**
  1.  **PuDHammer（读干扰放大）**：
      - **核心思想**：PiD的**多行激活**操作（包括连续激活CoMRA和同时激活SiMRA）本身就是一种强力的“锤击”模式。它通过同时或快速连续激活一个（SiMRA）或两个（CoMRA）相邻行，加剧了对中间未被访问的“受害行”的电磁干扰，从而以远低于传统RowHammer的访问次数就能诱导出比特翻转。
  2.  **IMPACT（时序信道攻击）**：
      - **核心思想**：PiM架构提供了用户空间应用程序绕过缓存、**直接**且**快速**访问主存的路径，共享的DRAM行缓冲状态（冲突或命中）会显著影响访问延迟。攻击者（发送方）可以通过在不同银行并行执行PiM指令（如RowClone）来编码信息（造成冲突=1，不造成冲突=0），而接收方则通过测量访问同一行时的延迟来解码信息。

### 论文的主要结论与发现

1.  **在真正的商用DRAM芯片上，可以高效地实现两个关键安全原语**：
    - SiMRA-TRNG：通过多行同时激活，可提供高达**16.05 Gb/s**的吞吐量，超过了当时最先进的QUAC-TRNG（可达1.99倍）。
    - SiMRA-PUF：能生成稳定且独特的设备签名，其**评估延迟比Frac-based PUF低5.75%**（2行激活情况下）。
2.  **PiD操作引入了亟待解决的严重安全挑战**：
    - 读干扰放大（PuDHammer）：通过SiMRA操作，引发第一个比特翻转所需的行激活次数（`HC_{first}`）比传统RowHammer最多可减少**158倍**，使得DRAM的可靠性问题急剧恶化。
    - 高速时序信道（IMPACT）：利用PiM提供的直接、快速内存访问，可以构建一个通信速率高达**14.8 Mb/s**的隐蔽信道，比先前最先进方案高**6.5倍**，并可用于高精度（96%）的侧信道攻击。
    - 现有防御机制（如针对RowHammer的PRAC，以及通用的缓存刷新或延迟抖动）在应对这类新型攻击时，会引入**巨大的性能开销**，亟需新的低开销解决方案。
3.  **最终结论**：未来的计算系统应将DRAM（以及一般意义上的内存）设计、使用和编程为一个**集计算、存储和安全于一体的基底**，而不仅仅是一个惰性存储设备。这需要从根本上改变研究和设计的思维模式。

### 不足与局限

1.  **技术局限**：
    - **PUF的可用性**：虽然SiMRA-PUF延迟更低，但其2行激活时的稳定性（Intra-Jaccard Index = 89.02%）略低于对比的Frac-based PUF。而稳定性最好的32行激活方案，延迟又显著增加（3.84倍）。
    - **攻击的通用性**：IMPACT攻击依赖特定的PiM指令集（如三星的HBM-PIM中的PnM指令或RowClone操作），其通用性可能受限于具体的PiM硬件架构。
2.  **实验覆盖**：
    - **DRAM代际**：实验主要基于**DDR4**芯片。论文中提到了DDR5引入了PRAC等新特性，但未对更先进的DRAM（如DDR5、HBM2e/3）进行实地测试，结果的可迁移性有待验证。
    - **系统级影响**：论文在评估防御机制开销时，使用了模拟器，而非全系统实际运行。例如，PuDHammer的PRAC性能开销是基于60个多核工作负载模拟，而IMPACT的防御评估是概念性的，缺乏实际硬件测试。
3.  **应用限制**：
    - **集成难度**：要将这些in-DRAM原语（如TRNG，PUF）集成到现有软件栈和操作系统中，需要修改内存控制器、驱动和分配器，工程复杂性高。
    - **安全权衡**：PUFs和TRNGs的生成操作依赖于违反时序规范，这本身就可能干扰系统中其他正在进行的正常内存访问，需要在安全性和系统可用性之间进行权衡。
    - **资源消耗**：生成这些原语需要占用DRAM资源（如特定行、bank），并由系统软件管理，可能减少可用内存容量或影响整体内存带宽分配。
