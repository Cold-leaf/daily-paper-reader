## 摘要
设备端大型语言模型（LLMs）的发展受到隐私保护和快速响应时间需求的驱动。移动设备上的高能耗数据传输使得内存中处理（PIM）成为一种有效的解决方案。由于严格的DRAM成本限制、电路板上有限的物理空间以及应用程序与LLMs之间的交互，CPU和PIM必须在共享内存空间中并发运行成为必要。然而，诸如存储体冲突和总线拥塞等挑战可能会出现，可能削弱PIM的性能和能效优势。为解决这一挑战，我们引入了COSM，一种协同调度框架，旨在促进移动平台上PIM和CPU任务的并发操作。我们的关键创新包括：1）一种低干扰的PIM控制接口，可在不干扰CPU内存访问的情况下生成最大数量的PIM命令；2）一种空闲感知调度方法，将PIM命令集成到CPU访问序列中的可用空闲时间窗口中。COSM不仅将PIM执行延迟对CPU隐藏，而且使PIM执行与数据传输重叠。在LLMs与移动工作负载（包括移动应用和计算密集型内核）并发执行的实验中，COSM相比基线调度方法将PIM吞吐量提高了最多2.8倍，同时CPU性能损失低于2.0%。

## Abstract
The development of on-device large language models (LLMs) is driven by the need for privacy and fast response times. Energy-intensive data transfer on mobile devices makes Processing-in-Memory (PIM) an effective solution. Due to stringent DRAM cost constraints, limited physical footprint on circuit boards, and the interaction between applications and LLMs, it is imperative for the CPU and PIM to operate concurrently within a shared memory space. However, challenges such as bank conflicts and bus congestion can arise, potentially diminishing the performance and energy benefits of PIM. To address this challenge, we introduce COSM, a cooperative scheduling framework designed to facilitate the concurrent operation of PIM and CPU tasks on mobile platforms. Our key innovations include: 1) a low-interference PIM control interface that generates the maximum number of PIM commands without disrupting CPU memory accesses; 2) an idleness-aware scheduling method that integrates PIM commands into available idle time windows within the CPU's access sequence. COSM not only hides PIM execution latency from the CPU, but also overlaps PIM execution with data transfer. Experiments on concurrent execution of LLMs and mobile workloads, including mobile applications and compute-intensive kernels, demonstrate that COSM improves PIM throughput by up to 2.8x compared to the baseline scheduling method with less than 2.0% CPU performance loss.

## 1. 核心问题与整体含义（研究动机与背景）

- **研究动机**：移动设备端部署大型语言模型（LLMs）是趋势（如隐私、低延迟），但传统CPU/NPU架构中数据搬运能耗占LLM推理总能耗的60%以上，导致续航和发热问题。处理-内存（PIM）通过在DRAM芯片内部集成计算单元绕过外部带宽瓶颈，降低能耗、提高内部带宽利用率，因此被视作有前景的方案。
- **核心问题**：在共享内存的CPU-PIM混合系统中，CPU和PIM并发执行时会出现存储体冲突（bank conflicts）和总线拥塞，导致PIM性能收益被削弱。具体表现为：PIM执行命令会阻塞CPU访问；CPU介导的数据传输（如输入/输出搬运）会抢占CPU任务的带宽，导致CPU性能严重下降（最高80%）。
- **整体含义**：需要一种既能最大化PIM内部带宽利用、又能最小化CPU访问干扰的协同调度框架，以平衡移动设备上LLM推理加速与交互式应用体验之间的矛盾。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：基于“CPU工作负载对外部带宽敏感、PIM工作负载对内部带宽敏感”的互补特性，提出一种“空闲感知（idleness-aware）”协同调度框架（COSM），将PIM命令（包括执行和数据传输）安排到CPU访问序列间的空闲时间窗口中，同时通过低干扰的PIM控制接口实现精确抢占。

- **关键技术细节**：
  1. **低干扰PIM控制接口**：
     - **可抢占的PIM执行命令（Preemptable PIM Execution Commands）**：设计PIM_Exec命令让PIM单元连续执行多列（指令长度nPTL可配置，默认128周期），避免命令总线饱和；同时引入PIM_Pause命令，可在列边界处快速暂停PIM执行，释放bank和总线供CPU访问，恢复时从暂停点继续执行。
     - **带宽解耦的数据传输命令（Bandwidth-Decoupled Data Transfer Commands）**：将传统CPU介导数据传输拆分为外部传输（PIM_RdBuf/PIM_WrBuf，占用外部总线）和内部传输（PIM_LdBuf/PIM_StBuf，占用内部bank带宽），允许独立调度，避免连续锁定总线。
  2. **空闲感知内存调度**：
     - **空闲时间窗口估算器（IWE）**：基于FR-FCFS策略预测每个bank和总线的最早可用服务时间，从而识别两类空闲窗口：①bank级空闲（如CPU行激活前的延迟）；②总线级空闲（CPU请求间空隙）。算法1给出了估算流程。
     - **PIM调度器**：在重叠执行阶段（overlapped execution phase），将PIM工作负载划分为无关的tile，优先调度外部总线命令（PIM_RdBuf/WrBuf），其次内部传输命令，最后是PIM执行命令；并引入PIM_Barrier命令保证tile间数据依赖。
  3. **架构实现**：在内存控制器中增加PIM执行队列（PEQ）、PIM读/写队列（PRWQ）、PIM调度器、空闲窗口估算器和命令仲裁器，采用严格优先级：CPU请求 > PIM_Pause > PIM命令。
