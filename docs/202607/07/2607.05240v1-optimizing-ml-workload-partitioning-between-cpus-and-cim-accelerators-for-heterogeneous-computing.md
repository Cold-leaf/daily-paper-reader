## 摘要
存内计算（CIM）加速器在内存中执行矩阵向量乘法（MVM），使其成为机器学习（ML）工作负载的有力解决方案。然而，现有的用于CIM加速器的ML工作负载划分方法并未充分考虑电阻式随机存取存储器（RRAM）的约束，如有限的内存、高写入延迟和有限的耐久性。它们还忽略了并行性、低级别架构效应以及中央处理器（CPU）作为补充计算资源的作用。为了解决这些限制，我们提出了一种基于整数线性规划（ILP）的异构CPU-CIM系统工作负载划分框架。该框架在RRAM约束下最小化端到端推理延迟，捕获并行性，并融合经验性能分析与分析模型。使用我们的框架，异构CPU-CIM执行相比边缘CPU上的纯CPU执行实现了高达30.9倍的加速，相比高性能CPU实现7.3倍的加速。设计空间探索（DSE）为未来的CIM加速器提供了进一步的设计见解。

## Abstract
Computing-in-Memory (CIM) accelerators execute Matrix-Vector Multiplications (MVMs) in memory, making them a compelling solution for Machine Learning (ML) workloads. However, existing ML workload partitioning approaches for CIM accelerators do not fully account for Resistive Random Access Memory (RRAM) constraints such as limited memory, high write latency, and limited endurance. They also neglect parallelism, low-level architectural effects, or the Central Processing Unit (CPU) as a complementary compute resource. To address these limitations, we propose an Integer Linear Programming (ILP)-based workload partitioning framework for heterogeneous CPU-CIM systems. It minimizes end-to-end inference latency under RRAM constraints, captures parallelism, and combines empirical profiling with analytical models. Using our framework, heterogeneous CPU-CIM execution achieves speedups of up to 30.9x over CPU-only execution on an edge CPU and 7.3x over a high-performance CPU. A Design Space Exploration (DSE) yields further design insights for future CIM accelerators.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：当前基于电阻式随机存取存储器（RRAM）的存内计算（CIM）加速器在执行机器学习（ML）推理时，面临有限存储容量、高写入延迟和有限耐久性等约束，难以将整个模型静态映射到CIM上。同时，现有ML工作负载划分方法大多将设备视为黑箱，忽略低层次架构特性、并行性，以及将CPU作为互补计算资源的潜力，导致系统资源利用不足。
- **核心问题**：如何在一个由主机CPU和RRAM-based CIM加速器组成的异构系统中，实现ML推理工作负载的最优静态划分，以最小化端到端推理延迟，同时满足CIM的存储预算和算子支持约束。
- **整体含义**：本文提出一种基于整数线性规划（ILP）的框架，通过融合经验分析与解析模型，系统性地解决CPU-CIM间的协同执行问题，为未来CIM加速器的设计提供指导。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将ML工作负载划分问题建模为ILP，在给定CIM存储容量（交叉开关数量）和算子支持约束下，最小化端到端推理延迟。框架由三部分组成：
  1. **混合延迟表征**：
     - 对主机CPU：使用ONNX Runtime实测每个算子的执行延迟（多次迭代、取中位数），以捕获真实的架构和优化效果。
     - 对CIM加速器：建立分析模型，包括负载、MVM计算和存储延迟。MVM延迟 = 输入向量数 × 单次MVM时间；负载/存储延迟取决于数据量和内部总线带宽。
     - 跨设备传输延迟：通过共享总线带宽和输出数据量计算。
  2. **ILP公式**：
     - 变量：每个算子分配决策（0=CIM, 1=CPU）；辅助变量（不同设备间的传输标志；CPU上并行算子的顺序）。
     - 目标：最小化执行时间（makespan），等于所有出口节点完成时间的最大值。
     - 主要约束：
       - 每个算子的开始时间 ≥ 所有前驱的完成时间 + 必要的传输延迟。
       - 当两个无依赖的算子都分配到CPU时，需引入顺序约束防止同时执行（实际是串行化，但原文通过二进制顺序变量模拟CPU上的抢占）。
       - CIM资源约束：所有分配到CIM的算子所需交叉开关总数不能超过可用数量（根据权重矩阵维度与交叉开关尺寸计算）。
  3. **设计空间探索（DSE）**：通过改变交叉开关数量、MVM延迟等参数，分析系统性能敏感度。

### 6. 论文的主要结论与发现

- **加速效果**：
  - 在ARM边缘CPU上，异构执行达到30.9倍加速（ResNet-18, 50ns MVM, 100+交叉开关）；在高性能x86 CPU上达到7.3倍加速。
  - MobileNetV2加速有限（因深度可分离卷积利用率低）；YOLOv5n达到3.1倍（ARM）和1.9倍（x86）。
- **资源饱和现象**：当交叉开关超过100个后，加速比饱和，因为所有计算密集算子已全部分配到CIM。
- **MVM延迟影响**：高MVM延迟（>500ns）使CIM加速无效，尤其在高性能CPU上。
- **ILP求解可扩展性**：线性/稀疏分支架构（ResNet, MobileNetV2）解得快（<0.01s）；分支复杂（YOLOv5n）导致变量和约束激增，求解时间可达数百秒，但解的质量仍可接受（MIP gap不大）。

### 7. 优点

- **方法完整**：将CPU经验 profiling与CIM分析模型结合，混合方法兼具精度与灵活性；ILP公式显式处理并行性、设备间传输、CIM容量约束。
- **实用性**：权重静态一次编程，符合RRAM耐久性要求；框架可直接用于不同CPU/CIM平台。
- **设计空间探索**：揭示交叉开关数量和MVM延迟对性能的影响，为未来CIM架构提供量化指导。
- **实验设置合理**：支持多种主流CNN架构，覆盖边缘和高端CPU场景。

### 8. 不足与局限

- **缺乏与现有方法的公平对比**：未与Neurosurgeon、COMPASS、CIM-MLC等经典划分方法比较，难以凸显本框架的优越性。
- **忽略多核CPU并行**：假设单核CPU，实际推理可在多核上并行，低估CPU性能；且CPU上的算子顺序约束不够精确（实际多核可并行多个算子）。
- **CIM模型简化**：假设GPEU操作完全流水线化，未考虑激活、池化等可能造成气泡；未模拟ADC/DAC能耗与精度损失。
- **ILP求解时间**：对复杂分支网络（YOLOv5n）求解时间接近10分钟，不适合在线动态划分；缺少启发式或混合方法。
- **仅推理场景**：未讨论训练或持续学习中的重编程开销。
- **未考虑多张量大模型的典型场景**：如Transformer（BERT）等可能更适合CIM加速，但未评估。
